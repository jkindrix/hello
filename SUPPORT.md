# Getting help

| Question type | Channel |
| --- | --- |
| Reproducible bug, build failure, test failure | [Open an issue](https://github.com/jkindrix/hello/issues/new/choose) using the **Bug report** template. |
| Feature request or API proposal | [Open an issue](https://github.com/jkindrix/hello/issues/new/choose) using the **Feature request** template. |
| Suspected security vulnerability | [Open a private security advisory](https://github.com/jkindrix/hello/security/advisories/new). Do not file public issues for security problems. |
| Usage questions, "how do I…", design discussion | [GitHub Discussions](https://github.com/jkindrix/hello/discussions). |
| Code of Conduct concerns | See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md). |

Before opening anything, please check existing issues and discussions — a
quick search saves both of us time, and adding to an existing thread is
usually more productive than starting a new one.

When you do open an issue, the templates ask for the information that
typically determines whether the report is reproducible. Filling them in
isn't busywork; missing fields are the most common reason a report sits
unanswered.
